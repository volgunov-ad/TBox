1. Структура входных данных (FuelEntry)
   При каждой фиксации заправки необходимо передать объект FuelEntry со следующими параметрами:
   odometerKm: (Double) Общий пробег.
   engineHours: (Double) Наработка двигателя (моточасы).
   sensorBefore: (Double) Показания датчика в литрах ДО заправки.
   sensorAfter: (Double) Показания датчика в литрах ПОСЛЕ заправки.
   litersByCheck: (Double) Объем заправленного топлива по чеку.
   ambientTemp: (Double) Температура воздуха в момент заправки (для термокомпенсации).
2.    Ключевые механизмы
      Калибровка зон (CalibrationStore)
      Бак программно разделен на сегменты (по умолчанию 5 зон). Для каждой зоны ведется накопительный учет:
      Реальный объем (из чеков, приведенный к +15°C).
      Сенсорный объем (разница показаний датчика).
      По их отношению вычисляется локальный коэффициент коррекции (K_local).
      Доверительная модель (Confidence)
      Для исключения влияния случайных заправок используется порог зрелости (maturityThreshold = 80л).
      Если данных в зоне мало, результат смешивается с глобальным средним коэффициентом всего бака.
      Итоговая формула коэффициента:
      K_final = (K_local * Confidence) + (K_global * (1 - Confidence))
      Эквивалентный расход
      Для корректного отображения расхода при длительных простоях/прогревах используется эквивалентный пробег:
      Расход (л/100км) = Литры / (Дистанция + (Моточасы * 30)) * 100
      (где 30 — настраиваемый коэффициент эквивалентности часа простоя к километрам).

## 5. Нюансы и советы по эксплуатации
* **Первая калибровка:** Система наиболее эффективна, если первые 2-3 заправки будут производиться "до полного" с максимально низкого уровня. Это сразу "прошьёт" все зоны бака достоверными данными.
* **Мертвые зоны:** Если датчик (поплавок) замер на максимальном значении (например, 48л из 50л), система будет выдавать уверенность ниже 100%, так как реальный объем в горловине физически не может быть измерен датчиком.
* **Сброс данных:** Для полного обнуления обучения достаточно удалить файл `calibration.json`. Это может потребоваться при замене топливного насоса или датчика уровня.
* **Температура:** Если датчик температуры улицы недоступен, рекомендуется передавать константу `15.0`, чтобы отключить термокомпенсацию и работать по "сырым" данным.
* * 
* * Настройка строгости фильтра: Рекомендуемое значение maxDeviationPercent — от 0.1 (строгий) до 0.3 (умеренный). Слишком низкое значение может привести к тому, что заправки на неровной поверхности будут ошибочно отклоняться.
* * Количество зон: Оптимально использовать 5–10 зон. Увеличение этого числа требует большего количества заправок для достижения 100% уверенности (Confidence) по всему баку.
* * ВАЖНО!!! Если данные о моточасах не будут поступать или будут дублироваться, отчет по расходу просто будет игнорировать простой и считать «чистую классику» (литры на километры).
* * При первом запуске на новом устройстве (без файла JSON) рекомендуется выполнить 2-3 полных цикла заправки для инициализации базовых коэффициентов всех зон бака
