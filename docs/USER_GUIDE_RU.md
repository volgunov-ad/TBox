# Руководство пользователя TBox Monitor

Приложение **TBox Monitor** предназначено для обмена данными между головным устройством (ГУ) Jetour Dashing и блоком TBox: мониторинг сети, CAN, геолокации и отправка служебных команд во внутренние программы TBox.

Документ из **трёх частей**: (1) **пошаговая работа** с интерфейсом; (2) **ARM/MCU и логика перезагрузок** внутри TBox; (3) **таблицы опций** в приложении и безопасность.

---

## Часть 1. Пошаговые инструкции

### 1.1. Первый запуск и проверка связи

1. Установите приложение и откройте его на ГУ.
2. Примите запрошенные разрешения, если планируете использовать соответствующие функции (хранилище для файлов, геолокация, уведомления для виджета музыки и т.д.).
3. В левом меню откройте **«Настройки»**, найдите блок **«Получение данных от TBox»** и включите **«Получать данные CAN»** и **«Получать данные о геопозиции»**, если вам нужны вкладки «Данные авто» и «Геопозиция» и связанные плитки.
4. Вернитесь в меню, откройте **«Модем»** и дождитесь появления строк с данными.
5. Разверните подписи в боковом меню (иконка вверху полоски меню) и посмотрите внизу: строка о **подключении TBox** должна быть зелёной, когда связь с блоком есть.

### 1.2. Как ходить по разделам

1. Слева — **меню**. Нажмите иконку **открыть/закрыть** вверху полоски, чтобы показать или скрыть **названия** пунктов.
2. Нажмите **«Главная»** — откроется **главный экран** (обои и панели с плитками), меню спрячется.
3. Чтобы снова открыть список разделов, на главном экране нажмите **плавающую кнопку с иконкой разделов** (её можно перетаскивать по экрану).
4. Любой другой пункт меню открывает **вкладку** справа (Модем, Плитки, Настройки и т.д.).

**Порядок пунктов** после «Главная»: Модем → AT-команды → Геопозиция → Данные авто → Поездки → Настройки → Настройки главного экрана → Журнал → Информация → CAN → Плитки.

Пункты **AT-команды**, **Журнал** и **CAN** появляются только после включения **«Экспертный режим»** в «Настройках». Без экспертного режима этих строк в меню нет.

### 1.3. Настроить главный экран с нуля

1. Меню → **«Настройки главного экрана»**.
2. (По желанию) Включите **«Автоматически открывать главный экран при запуске»**, если после загрузки ГУ хотите сразу видеть главную страницу приложения.
3. **Обои:** подпись про светлую тему → **«Выбрать изображение»** → укажите файл; для удаления обоев — **«Удалить»** (когда картинка задана). То же для **тёмной** темы.
4. Включите или выключите переключатель **масштаба фона (Crop)** — картинка на весь экран с обрезкой или вписывание целиком (см. подпись на экране).
5. В блоке **«Панели главного экрана»** нажмите **«Добавить»**.
6. В **выпадающем списке** выберите созданную панель. В поле **имени** введите удобное название и нажмите кнопку **переименования** (она активна, когда имя изменилось).
7. Включите **«Показывать панель на главном экране»**.
8. При необходимости задайте **число строк** и **столбцов** плиток, **открытие программы одиночным нажатием** и **индикатор отключения TBox**.
9. Поля **ширина, высота, позиция X и Y в %** задайте так, чтобы панель занимала нужную область (можно начать с значений по умолчанию и потом подправить).
10. Меню → **«Главная»** — убедитесь, что панель видна.
11. **Переместить или изменить размер панели на экране:** удерживайте панель и перетаскивайте; размер меняйте, как показывает интерфейс при удержании/за край (следуйте подсказкам на экране).
12. **Назначить плитку:** на главном экране **долго нажмите** ячейку → в диалоге выберите, что показывать → при необходимости откройте **«Дополнительно»** (цвета по светлой/тёмной теме, масштаб, скругление; для музыки — плееры) или **«Вся панель»** (имя, строки/столбцы, **открытие программы одиночным нажатием**, индикатор TBox) → сохраните. Повторите для остальных ячеек.
13. **Одиночное нажатие на плитку на главном экране:** при включённой опции **«открывать окно программы…»** по тапу открывается только вкладка **«Поездки»**, если на плитке выбран виджет **активной поездки** (или упрощённый вариант); для остальных плиток эта опция сейчас **ничего не делает**.

**Вторая и следующие панели:** снова **«Добавить»** в блоке панелей; в списке переключайтесь между панелями и настраивайте каждую отдельно. Лишнюю панель удалите кнопкой **«Удалить»** (выбранная в списке).

### 1.4. Настроить вкладку «Плитки»

1. **«Настройки»** → блок **«Настройки экрана Плитки»**.
2. Выберите **количество строк** и **столбцов** — столько будет ячеек на вкладке «Плитки».
3. При желании включите **«Показывать графики…»** на плитках.
4. Меню → **«Плитки»**.
5. **Долгое нажатие** на ячейку → выбор данных → **«Дополнительно»** при необходимости → сохранение.
6. **Короткое нажатие** по плитке выполняет действие этого виджета (запуск приложения, команды на плитке и т.д.).

Для **музыки** система может запросить доступ к **уведомлениям**. Для плитки **стороннего виджета** следуйте кнопкам выбора виджета в диалоге.

### 1.5. Включить плавающую панель поверх других приложений

1. **«Настройки»** → **«Настройки плавающих панелей»**.
2. Нажмите **«Добавить»** — в списке появится новая панель.
3. В **выпадающем списке** выберите панель, задайте **имя** и нажмите кнопку **переименования**, если меняли название.
4. Включите **«Показывать плавающую панель»**. Если появится запрос Android, откройте **настройки** по кнопке в диалоге и разрешите **«Отображение поверх других окон»** для TBox Monitor.
5. Установите **строки** и **столбцы** сетки.
6. Введите **ширину и высоту окна в пикселях** и координаты **X, Y** (в полях ввода минимум 50 для ширины и высоты). При необходимости уменьшите или увеличьте числа, если окно неудобно расположено.
7. Выйдите на рабочий стол или другое приложение: должно появиться **окно** с плитками.
8. **Назначить плитки:** в этом окне **долго нажмите** ячейку — откроется диалог выбора данных; при необходимости откройте **«Вся панель»** (строки/столбцы, индикатор TBox, **открытие программы одиночным нажатием** — как в настройках плавающей панели).
9. **Переместить или изменить размер окна:** перетаскивайте окно и тяните за край изменения размера (как показывает интерфейс панели).

**Несколько плавающих панелей:** добавьте ещё одну кнопкой **«Добавить»**. На экране одновременно показываются **все панели, у которых в конфигурации включён показ** (каждая со своим положением и размером). Чтобы настроить **размер в пикселях и сетку** для конкретной панели, сначала **выберите её в выпадающем списке** в этом блоке настроек — изменения применяются к выбранной панели.

### 1.6. Включить цветные индикаторы на плитках

1. **«Настройки»** → **«Настройки виджетов для Overlays»**.
2. Включите **индикатор подключения TBox**: **красный** — нет данных от службы, **жёлтый** — нет связи с TBox, **зелёный** — связь есть.
3. Включите **индикатор геопозиции** (нужно включённое получение геоданных): **красный** — нет фикса, **жёлтый** — расхождение скорости со спутниками, **зелёный** — норма.

### 1.7. Вкладка «Модем» — пошагово

1. Меню → **«Модем»** — список обновится (служба запрашивает состояние).
2. Прочитайте блок **«Данные модема и SIM»** (IMEI, ICCID, IMSI, оператор) — так проверяете, видит ли TBox карту и сеть.
3. В блоке **«Данные подключения»** смотрите **CSQ**, **регистрацию**, **SIM**, **сеть**, строку **APN**, **время подключения**.
4. Пролистайте **APN 1** и **APN 2** — IP, шлюз, DNS, время изменения.
5. Внизу нажмите **«Включен»**, **«Режим полёта»** или **«Выключен»** по необходимости. После нажатия кнопки кратко блокируются от повторного нажатия.

### 1.8. Вкладка «Геопозиция» — пошагово

1. В **«Настройках»** включите **«Получать данные о геопозиции»**.
2. Меню → **«Геопозиция»**.
3. Смотрите **«Последнее обновление»** (обновление экрана) и **«Последнее изменение»** (изменились ли сами данные).
4. Строки **«Фиксация»** и **«Правдивость»** (да/нет) — пригодность координат.
5. Ниже — координаты, спутники, скорость, курсы, UTC, **«Сырые данные»**.

Если пусто — проверьте связь с TBox и настройку получения геоданных (шаг 1).

### 1.9. Вкладка «Данные авто» — пошагово

1. В **«Настройках»** включите **«Получать данные CAN»**.
2. Меню → **«Данные авто»** — прокрутите список.
3. Пустые поля для части параметров возможны: не всё доступно на каждом авто или в каждый момент.

### 1.10. Вкладка «Поездки» — пошагово

1. Меню → **«Поездки»**.
2. Управляйте **текущей поездкой** кнопками на экране (старт/стоп — как отображено в интерфейсе).
3. Просматривайте список, при необходимости отмечайте **избранное**.
4. Чтобы изменить правило **разбиения поездки по времени**, откройте **«Настройки»** → **«Прочее»** → параметр **минут разделения**.
5. Для сохранения данных используйте кнопки **сохранения в файл** на экране поездок, если они есть.

### 1.11. Вкладка «Информация» — пошагово

1. Убедитесь, что TBox **подключён** (индикатор в меню).
2. Нажмите **«Запросить информацию из TBox»**.
3. Дождитесь заполнения полей версий и VIN. Кнопка после запроса ненадолго недоступна.
4. Значение **HW** используйте вместе с **таблицей в части 3** для опций предотвращения перезагрузки.

### 1.12. Включить автоматический перезапуск модема или перезагрузку TBox

1. **«Настройки»** → **«Настройки контроля сети»**.
2. Включите **«Автоматический перезапуск модема»**, если нужно, чтобы при пропаже сети приложение само перезапускало модем (интервалы указаны в подписи под переключателем).
3. Включите **«Автоматическую перезагрузку TBox»** только если нужна перезагрузка блока после неудачного перезапуска модема (доступно при включённом шаге 2; интервалы — в подписи).

### 1.13. Уменьшить перезагрузки TBox из-за APP/SWD/MDC (пошагово)

1. Меню → **«Информация»** → **«Запросить информацию из TBox»** → запомните **HW** и наличие SIM.
2. **«Настройки»** → блок **«Настройки предотвращения перезагрузки»**.
3. **Выключите все** переключатели в этом блоке.
4. По **таблице в части 3** этого документа включите **только** указанные там опции (SUSPEND/STOP для APP, при необходимости SWD, MDC).
5. Если приложение показало предупреждение о **ручной перезагрузке TBox** — выполните перезагрузку TBox вручную, как рекомендует диалог.

Подробности про SWD, APP и риски — в **части 2**.

### 1.14. Ручная перезагрузка TBox из приложения

1. **«Настройки»**, прокрутите вниз.
2. Нажмите **«Перезагрузка TBox»** (кнопка активна только при связи с TBox).
3. Не нажимайте повторно, пока кнопка снова не станет активной.

Это перезагрузка **ARM** (см. часть 2).

### 1.15. Резервная копия

1. **«Настройки»** → **«Резервная копия»**.
2. **Экспорт:** **«Экспорт JSON»** → подтвердите сохранение (например, в **Загрузки**). Обои в файл **не входят**.
3. **Импорт:** **«Импорт JSON»** → прочитайте предупреждение → **«Выбрать файл»** → укажите сохранённый JSON. Текущие настройки и поездки **заменятся**. Плитки со **сторонними виджетами** нужно настроить заново.
4. После импорта при необходимости снова задайте **обои** в **«Настройках главного экрана»**.

### 1.16. Экспертный режим, журнал, AT, CAN

1. **«Настройки»** → **«Прочее»** → включите **«Экспертный режим»**, подтвердите предупреждение.
2. В меню появятся **«Журнал»**, **«AT-команды»**, **«CAN»**.
3. **Журнал:** откройте пункт → введите **фильтр** (не меньше 3 символов), если нужно → **«Сохранить в файл»** / сохранение в Загрузки через диалог.
4. **AT-команды:** введите команду → отправьте; результат смотрите в журнале и в поведении модема.
5. **CAN:** просмотрите список ID → при необходимости сохраните в файл. В **«Настройках»** в экспертном режиме задайте **число сохраняемых CAN-фреймов**.

### 1.17. Подмена геолокации (экспертный режим)

1. Включите **экспертный режим** (см. 1.16).
2. Включите **«Подменять системные данные о геопозиции»**.
3. Если условия не выполнены, нажмите **ссылку с требованиями** и по шагам включите **фиктивные местоположения** и разрешения в настройках разработчика Android.

### 1.18. Задать объём бака для расчётов по топливу

1. **«Настройки»** → **«Прочее»**.
2. Найдите параметр **объёма бака (литры)** и введите значение в допустимом диапазоне (на экране указаны пределы).
3. Вернитесь на плитки с уровнем топлива и проверьте отображение при необходимости.

## Часть 2. Устройство TBox: ARM и MCU

На одной плате TBox условно два узла:

- **ARM** — встроен в модем 4G с GPS (модули Neoway N720 или N725 в зависимости от версии аппаратной части TBox). На нём работает ОС Linux; к нему ГУ подключается по USB. Все перечисленные ниже программы (APP, SWD, MDC, CRT и др.) выполняются на ARM. Перезагрузка ARM **не влияет** на остальные блоки автомобиля.

- **MCU** — связан с **тремя** шинами CAN, собирает данные и передаёт их в ARM по внутренней шине (SPI), принимает команды от ARM и выдаёт их в CAN. За обмен ARM ↔ MCU отвечает программа **CRT**.

Кнопка **«Перезагрузка TBox»** в меню **Настройки** отправляет в CRT команду на перезагрузку **только ARM** — это штатный и безопасный сценарий. Перезагрузка MCU в отдельную функцию приложения не выносилась и в общем доступе не предлагалась; при экспериментах с перезагрузкой MCU на приборной панели могла появляться ошибка, которая затем пропадала.

## Программы внутри TBox (кратко)

Версии прошивки отличаются; поведение отдельных процессов может немного меняться.

### SWD (сторож)

Следит за общим состоянием OpenLinux, за другими программами (APP, MDC, LOC и т.д.) и за сетью. Если какая-то программа «падает», SWD пытается её снова запустить; при повторных сбоях инициирует перезагрузку TBox.

- На **HW 0.0.5** при сценарии с полной остановкой APP (команда **STOP** в приложении — см. ниже) типично происходит **одна** перезагрузка, после чего SWD перестаёт циклически поднимать процесс и гонять перезагрузки.
- На **остальных** рассматриваемых версиях этот цикл может **зацикливаться** и приводить к постоянным перезагрузкам.

Дополнительно перезагрузки могут инициироваться из-за проблем в мобильной сети и из-за аномалий системного таймера Linux.

Команда **SUSPEND** для SWD (в настройках: **автоматическая отправка команды SUSPEND приложению SWD**) приостанавливает эти механизмы; TBox обычно **не должен** из-за SWD перезагружаться. При этом отключается встроенная заводская защита от зависаний — используйте осознанно.

В приложении также есть опция **PREVENT RESTART** для SWD — она отключает часть проверок (сеть, аномалии), которые тоже могут приводить к лишним перезагрузкам.

### APP

Связана с облачным MQTT и передачей данных с камер; при невозможности установить соединение (в т.ч. при установленной SIM) может инициировать перезагрузку TBox **примерно раз в 30 минут**.

- На **0.0.4** и **0.0.5** для снятия этих перезагрузок обычно достаточно **SUSPEND** для APP.
- На **0.0.1** перезагрузки могут продолжаться; там нужна команда **STOP** для APP, но **только вместе с SUSPEND для SWD** — иначе возможны частые перезагрузки из-за поведения SWD.

### MDC

Получает от модема данные о сотовой сети и интернете. Внутри есть логика, которая тоже может приводить к перезагрузке (детали могут отличаться по версии). **SUSPEND** для MDC обычно убирает такие перезагрузки, но не всем это нужно — при необходимости пробуйте по месту. В настройках приложения указано, что при установленной SIM карте опции **SUSPEND/STOP для MDC** не рекомендуется включать без веской причины.

### CRT, HUM, LOC, GATE и др.

Отвечают за отдельные функции. **CRT**, в частности, принимает от SWD, APP, MDC и др. запросы на перезагрузку и выполняет перезагрузку системы TBox (на практике — сценарий с перезагрузкой ARM при использовании кнопки в приложении). **CRT отключать нельзя**: он также участвует в получении CAN-данных и других задачах.

## Часть 3. Опции предотвращения перезагрузки и таблицы

## Настройки предотвращения перезагрузки в TBox Monitor

Команды **SUSPEND** и **STOP** действуют до следующей перезагрузки TBox: после перезагрузки процессы снова стартуют в обычном режиме. **TBox Monitor** при включённых опциях может повторно отправлять команды. Всё **обратимо**: выключите опции и перезагрузите TBox вручную — поведение вернётся к заводскому.

При **смене набора** опций SUSPEND/STOP приложение предупреждает, что может потребоваться **ручная перезагрузка TBox** — следуйте подсказке на экране.

Имена опций в интерфейсе (раздел настроек про предотвращение перезагрузки):

| В приложении | Смысл |
|--------------|--------|
| Автоматическая отправка команды **SUSPEND** приложению **APP** | Приостановка APP |
| Автоматическая отправка команды **STOP** приложению **APP** | Полная остановка APP |
| Автоматическая отправка команды **SUSPEND** приложению **SWD** | Приостановка сторожа SWD |
| Автоматическая отправка команды **SUSPEND** приложению **MDC** | Приостановка MDC |
| Автоматическая отправка команды **STOP** приложению **MDC** | Остановка MDC |
| Автоматическая отправка команды **PREVENT RESTART** приложению **SWD** | Отключение части проверок SWD (сеть и др.) |

Рекомендация: для выбранного сценария включайте **только** нужные строки из таблицы ниже и **отключайте остальные** опции этого блока (включая PREVENT RESTART и MDC), если для вашей версии они не указаны.

## Что включить в зависимости от версии HW TBox и SIM

Узнать версию можно в диагностике TBox (запрос информации из TBox), в т.ч. поле версии HW.

| Версия HW TBox | SIM | Рекомендуемые опции в приложении | Примечание |
|----------------|-----|----------------------------------|------------|
| **0.0.5** | Нет | **SUSPEND** для APP | Остальные опции предотвращения перезагрузки — выкл. |
| **0.0.5** | Есть | **STOP** для APP | Остальные — выкл. Ожидайте **однократную** перезагрузку TBox — это нормальное проявление логики SWD на этой версии. |
| **0.0.4** | Нет или есть | **SUSPEND** для APP | Остальные — выкл. |
| **0.0.1** | (любая) | **STOP** для APP **и** **SUSPEND** для SWD | Остальные — выкл. Если перезагрузки всё ещё бывают, можно **дополнительно** попробовать **SUSPEND** для MDC. |

Если для вашей комбинации HW/SIM поведение отличается, ориентируйтесь на логи приложения и при необходимости сузьте набор включённых опций.

## Прочие настройки, влияющие на перезагрузки

В том же разделе настроек есть **автоматическая перезагрузка TBox** и **автоматический перезапуск модема** — это **сценарии самого TBox Monitor** при потере сети, а не заводская логика SWD/APP. При отладке «лишних» перезагрузок имейте в виду, что эти тумблеры могут добавлять свои перезапуски.

---

## Где узнать версии программ в TBox

В приложении доступен запрос информации из TBox (в т.ч. версии APP, CRT, LOC, MDC, SWD, ПО и HW). Используйте это для проверки, что вы настроили правильную ветку по таблице выше.

---

## Краткий чеклист безопасности

1. Не включайте **STOP APP** без **SUSPEND SWD** на **0.0.1** и **0.0.4** — риск непрерывных перезагрузок.
2. Не отключайте CRT — это не предусмотрено приложением и нарушит работу CAN и перезагрузок по команде.
3. Для возврата к заводскому поведению выключите опции предотвращения перезагрузки и при необходимости выполните перезагрузку TBox вручную.
